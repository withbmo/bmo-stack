# Security Policy

## Reporting a vulnerability

Please do not report security vulnerabilities through public GitHub issues.

Instead:

- contact the maintainers privately
- include a clear description of the issue
- include affected paths, components, and versions if known
- include reproduction steps or a proof of concept when safe
- explain the expected impact

If a private security contact address is added later, this file should be updated to point to it directly.

## What to include

Helpful reports usually include:

- vulnerability type
- affected component
- attack prerequisites
- proof of concept
- possible mitigations

## Disclosure expectations

- We prefer coordinated disclosure
- Please give maintainers reasonable time to investigate and patch
- Avoid publishing exploit details before a fix or mitigation is available

## Scope

This policy applies to:

- the application code
- starter templates
- project spec parsing and planning code
- local developer tooling included in this repository
