# Documentation patterns

Documentation quality compounds over time.

## Structure

- Start every page with purpose and audience.
- Keep code examples short and executable.
- Link related guides for faster onboarding.

## Ownership

- Assign maintainers to high-traffic docs.
- Track stale pages with periodic reviews.

## Final note

Docs are part of the product. Treat them with the same quality bar.
